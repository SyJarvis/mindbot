# 系统基础信息技能

## 使用目标
- 当用户需要了解当前运行环境的基础信息时，优先使用本技能组织回答。
- 回答时先给出简洁结论，再补充关键命令输出或文件信息。
- 尽量使用现有工具获取实时结果，不要凭空猜测系统信息。

## 推荐查询内容
- 操作系统与内核信息
- 当前工作目录或仓库位置
- Python 版本
- 工作区下的基础目录结构
- 某个配置文件或说明文件是否存在

## 推荐工具使用方式
- 使用 `exec_command` 获取实时系统信息，例如：
  - `pwd`
  - `python --version`
  - `uname -a`
  - `git status --short`
- 使用 `list_directory` 查看工作区目录结构。
- 使用 `file_info` 判断文件或目录是否存在以及类型。
- 使用 `read_file` 读取用户明确提到的配置文件或文档。

## 回答原则
- 如果用户只问一个简单问题，例如“当前目录是什么”或“Python 版本是多少”，只查询必要信息，不要过度展开。
- 如果用户询问“系统基本信息”，优先覆盖：
  - 操作系统
  - 当前目录
  - Python 版本
  - 仓库/工作区概况
- 如果用户要求引用依据，可以附上关键命令结果摘要。

## 安全与边界
- 不要主动输出完整环境变量或其他可能包含敏感信息的内容。
- 不要执行与用户问题无关的命令。
- 如果某个命令失败，应明确说明失败原因，并尝试使用其他安全工具补充信息。
